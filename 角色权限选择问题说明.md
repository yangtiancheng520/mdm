# 角色权限选择问题说明

## 问题描述
在角色管理中，选择权限时，只选择一级菜单下面的一部分权限后，保存后再次编辑，发现一级菜单下的所有权限都被选中了。

## 问题原因

### 原来的代码逻辑
```javascript
const checkedKeys = treeRef.value?.getCheckedKeys() || []
const halfCheckedKeys = treeRef.value?.getHalfCheckedKeys() || []

// 合并勾选节点和半选节点（父节点）
form.value.permissions = [...checkedKeys, ...halfCheckedKeys]
```

### 问题分析
1. 当用户只选择了部分子权限时，父节点会变成**半选状态**（half-checked）
2. 保存时，半选状态的父节点ID也被保存到了数据库
3. 再次编辑时，调用 `setCheckedKeys(form.value.permissions)`
4. Element Plus的树组件发现父节点ID在keys中，就会把该父节点下的所有子节点都勾选上

### 示例
假设权限树结构：
```
系统管理 (id: 1)
  ├─ 用户管理 (id: 2)
  ├─ 角色管理 (id: 3)
  └─ 权限管理 (id: 4)
```

用户操作：
1. 只勾选了"用户管理"和"角色管理"
2. 此时：
   - checkedKeys = [2, 3]
   - halfCheckedKeys = [1]  ← 父节点"系统管理"是半选状态
3. 保存到数据库：permissions = [2, 3, 1]  ← 包含了父节点ID
4. 再次编辑，setCheckedKeys([2, 3, 1])
5. 树组件发现父节点1在keys中，就把所有子节点2,3,4都勾选上了

## 解决方案

### 修复后的代码
```javascript
const checkedKeys = treeRef.value?.getCheckedKeys() || []
// 只保存叶子节点的权限，不保存父节点
const leafKeys = checkedKeys.filter(key => {
  const node = treeRef.value?.getNode(key)
  return node && node.isLeaf
})

form.value.permissions = leafKeys
```

### 修复原理
1. 只保存叶子节点（最底层的权限）的ID
2. 不保存父节点的ID
3. 再次编辑时，setCheckedKeys只设置叶子节点的keys
4. Element Plus会自动处理父子关系：
   - 如果所有子节点都被选中，父节点自动变成勾选状态
   - 如果部分子节点被选中，父节点自动变成半选状态

### 示例（修复后）
用户操作：
1. 只勾选了"用户管理"和"角色管理"
2. 此时：
   - checkedKeys = [2, 3]
   - leafKeys = [2, 3]  ← 只保存叶子节点
3. 保存到数据库：permissions = [2, 3]  ← 不包含父节点ID
4. 再次编辑，setCheckedKeys([2, 3])
5. 树组件正确显示：
   - "用户管理"和"角色管理"被勾选
   - "权限管理"未勾选
   - "系统管理"显示为半选状态（正确）

## Element Plus树组件的行为

### setCheckedKeys的特性
- 设置叶子节点的keys时，会自动计算父节点的状态
- 如果设置了父节点的key，会强制勾选该父节点下的所有子节点
- 因此，应该只设置叶子节点的keys，让树组件自动处理父节点状态

### 最佳实践
- **保存时**：只保存叶子节点的权限ID
- **回显时**：使用setCheckedKeys设置叶子节点的keys
- **显示时**：树组件会自动处理父节点的勾选/半选状态
