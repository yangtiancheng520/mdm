# 主数据状态流转与质量管理方案

## ✅ 已实现功能

### 1. 新状态体系

```typescript
DRAFT                    // 草稿（灰色）
PENDING                  // 审批中（橙色）
PENDING_QC              // 待质检（蓝色）
ACTIVE_QUALIFIED        // 已生效-合格（绿色）
ACTIVE_UNQUALIFIED      // 已生效-不合格（红色）
OBSOLETE                // 已作废（灰色）
```

### 2. 状态流转

```
草稿 (DRAFT)
  ↓ 提交审批
审批中 (PENDING)
  ↓ 通过审批
待质检 (PENDING_QC) ← 新增：审批通过后先进入待质检
  ↓ 执行质检
已生效-合格 (ACTIVE_QUALIFIED) ← 质检通过
已生效-不合格 (ACTIVE_UNQUALIFIED) ← 质检不通过
  ↓ 作废
已作废 (OBSOLETE)
```

### 3. 操作权限

| 状态 | 可用操作 |
|------|---------|
| 草稿 | 提交审批、编辑、删除 |
| 审批中 | 通过审批、驳回审批、查看 |
| 待质检 | 执行质检、查看 |
| 已生效-合格 | 查看、编辑、作废 |
| 已生效-不合格 | 查看、重新质检、作废 |
| 已作废 | 查看 |

## 🔄 业务流程

### 完整流程

1. **数据创建** → 草稿状态
2. **提交审批** → 审批中状态
3. **审批通过** → 待质检状态（⭐ 关键环节）
4. **执行质检** → 质检通过/不通过
5. **质检合格** → 已生效-合格
6. **质检不合格** → 已生效-不合格
7. **数据分发** → 仅已生效-合格的数据可以分发

### 质量检查环节

**触发方式：**
- ✅ 审批通过后自动触发（推荐）
- ✅ 手动执行质检
- ✅ 定期质检（定时任务）

**质检内容：**
- 完整性检查：必填字段是否完整
- 唯一性检查：关键字段是否重复
- 准确性检查：数据格式是否正确
- 一致性检查：关联数据是否匹配
- 值域检查：字段值是否在允许范围内

**质检结果：**
- 通过率 ≥ 80% → ACTIVE_QUALIFIED（合格）
- 通过率 < 80% → ACTIVE_UNQUALIFIED（不合格）

## 🚀 使用说明

### 前端操作

1. **提交审批**
   - 草稿状态 → 点击"提交审批"按钮
   - 数据进入"审批中"状态

2. **审批操作**
   - 审批中状态 → 点击"通过审批"按钮
   - 数据自动进入"待质检"状态
   - 提示："审批通过，数据已进入待质检状态"

3. **执行质检**
   - 待质检状态 → 点击"执行质检"按钮
   - 系统执行质检规则
   - 显示质检结果（通过率）
   - 自动更新为合格/不合格状态

4. **重新质检**
   - 已生效-不合格状态 → 点击"重新质检"按钮
   - 重新执行质检流程

---

## ⚠️ 注意事项

1. **强制质检**
   - 审批通过后必须经过质检
   - 无法跳过质检环节

2. **质检不合格**
   - 数据进入"已生效-不合格"状态
   - 可以重新质检
   - 可以直接作废

3. **数据分发**
   - 只有"已生效-合格"的数据可以分发
   - 不合格数据限制分发
