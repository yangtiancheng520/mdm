## 🔍 问题分析

### 问题1: 为什么000000120000000009失败？

**错误信息**: "基准计量单位不能更改"

**可能原因:**

1. **物料已存在于SAP中**
   ```
   SAP中已存在的物料信息:
   - 物料编码: 000000120000000009
   - 基本单位: ？？？ (未知)

   本次尝试创建/更新:
   - 物料编码: 000000120000000009
   - 基本单位: EA

   → 单位冲突！SAP不允许修改已存在物料的基本单位
   ```

2. **分发历史记录**
   ```
   第1次尝试: CREATE → 失败 (物料已存在，单位不同)
   第2次尝试: CREATE → 失败 (同样的错误)
   ```

### 问题2: 操作类型为什么是NULL？

从日志可以看到，第二次分发的操作类型是`NULL`，说明：

```
新逻辑：先尝试UPDATE → 失败 → 再尝试CREATE → 也失败
最终都没有成功，所以操作类型为空
```

## 💡 解决方案

### 方案1: 确认SAP中物料的基本单位

**步骤:**
1. 登录SAP系统
2. 使用事务代码 `MM03` (显示物料)
3. 输入物料编码: `000000120000000009`
4. 查看基本计量单位
5. 如果单位是 `KG` 或其他，而您需要 `EA`，则：
   - **无法修改**（SAP不允许修改基本单位）
   - 只能创建新的物料编码

### 方案2: 修改物料编码（推荐）

如果这是测试数据：
```
方案A: 删除SAP中的旧物料
  - 事务代码: MM06 (标记删除)
  - 然后重新分发

方案B: 使用新的物料编码
  - 创建新物料: 000000120000000010
  - 基本单位: EA
```

### 方案3: 调整数据以匹配SAP

```
如果SAP中的基本单位是 KG:
  - 修改MDM数据，将基本单位改为 KG
  - 重新分发（使用UPDATE模式）
```

